### ✅ 任务3：补充销售流程重试接口

**修改文件**: `/opt/workplace-ai-platform/api/app/routers/department_workspaces.py`

**新增API端点**:

#### 1. 重新生成单个销售项目
```
POST /api/department-workspaces/{team_id}/{department_key}/sales/regenerate/{item_id}
```

**请求体**:
```json
{
  "regenerate_by_member_id": "member_123",  // 可选
  "item_type": "lead",  // "lead" | "playbook" | "follow_up"
  "force": false  // 是否强制重新生成
}
```

**响应**:
```json
{
  "ok": true,
  "team_id": "team_123",
  "department": {...},
  "item_id": "lead_001",
  "item_type": "lead",
  "original_item": {...},  // 原始数据
  "new_item": {...},  // 重新生成的数据
  "generated_by": "model",  // "model" | "fallback"
  "warnings": [],  // 警告信息列表
  "attempts": 1  // 尝试次数
}
```

**功能说明**:
- 根据 `item_id` 查找原始数据
- 调用增强的 `sales_ai.generate_sales_launch_package()` 重新生成
- 保持原始ID不变，只更新内容
- 自动更新数据库中的 sales pipeline
- 支持三种类型：线索(lead)、剧本(playbook)、跟进计划(follow_up)

#### 2. 重新生成整个销售流程
```
POST /api/department-workspaces/{team_id}/{department_key}/sales/regenerate-all
```

**响应**:
```json
{
  "ok": true,
  "team_id": "team_123",
  "department": {...},
  "leads": [...],  // 所有重新生成的线索
  "playbooks": [...],  // 所有重新生成的剧本
  "follow_ups": [...],  // 所有重新生成的跟进计划
  "generated_by": "model",
  "warnings": [],
  "attempts": 2
}
```

**功能说明**:
- 完整重新生成所有销售启动包内容
- 使用最新的销售简报(sales_brief)作为输入
- 自动替换整个 sales pipeline
- 支持质量检查和重试机制

**新增数据模型**:
```python
class RegenerateSalesItemRequest(BaseModel):
    regenerate_by_member_id: str | None = None
    item_type: Literal["lead", "playbook", "follow_up"] = "lead"
    force: bool = False
```

## 技术细节

### 质量检查逻辑
- **最小长度检查**: need字段至少15字符，next_step至少10字符，opener至少20字符
- **内容重复度检查**: 独特词汇比例必须大于30%
- **占位符检查**: 拒绝包含"TODO"、"待补充"等占位符的内容
- **结构完整性检查**: 确保leads、playbooks、follow_ups都有生成

### 重试策略
- 默认最多重试3次
- 每次失败记录详细的错误信息
- 第3次失败后自动使用模板兜底
- 保证系统永远不会因为AI生成失败而中断

### 错误处理层级
1. **函数级**: sales_ai 内部的异常捕获和重试
2. **路由级**: 每个endpoint的try-catch处理
3. **全局级**: main.py的全局异常处理器

### 日志记录
- 所有请求都会记录：方法、路径、处理时间
- 所有错误都会记录：完整堆栈、错误ID、上下文信息
- 生成过程记录：尝试次数、失败原因、最终结果

---

## 性能影响

- **单次生成**: 如果质量不达标可能需要重试，最多增加2倍时间
- **首次成功率**: 取决于模型质量和prompt设计
- **最坏情况**: 3次重试失败后使用模板兜底，总时间约为单次生成的3倍
- **模板兜底**: 即时响应，零延迟
